1.2 布尔运行（Boolean algebra）

布尔运算符有&（AND）, |（OR）, !（NOT），xor()（exclusive OR）。需要注意的是||和&&是短路运算符，他们只会返回一个TRUE/FALSE值，通常用于控制流语句中。

1.2.1 在布尔运算中的NA

在布尔运算中，NA有点复杂，比如下面的例子，你乍看一眼会觉得不一致。

df <- tibble(
  x = c(TRUE, FALSE, NA)
)
df |> 
  mutate(
    and = x & NA,
    or = x | NA
  )
# A tibble: 3 × 3
  x     and   or   
  <lgl> <lgl> <lgl>
1 TRUE  NA    TRUE 
2 FALSE FALSE NA   
3 NA    NA    NA

一个非常好的帮助我们理解的方法是：我们可以把NA想象成既可以是TRUE也可是FALSE的值，那么TRUE & NA的结果可以是TRUE & TRUE（TRUE）或者TRUE & FALSE（FALSE），所以最终结果是NA。同样的，FALSE | NA的结果可以是FALSE | TRUE（TRUE）或者FALSE | FALSE（FALSE），所以最终结果FALSE。NA 遇到任何值任何运算都是NA。

1.2.2 %in% 操作符

%in%操作符是一个非常有用的工具，可以用来检查一个值是否在一个向量中。它的语法是：x %in% y，返回一个长度与x相同的逻辑向量，表示x中的每个元素是否在y中出现过。注意，NA在%in%和布尔比较中结果不一致，NA %in% c(NA, 1)返回TRUE，而NA == NA返回NA。

1.2.3 对逻辑向量进行统计汇总（Logical summaries）

在R里，TRUE会被当做1，FALSE会被当做0。因此，我们可以对逻辑向量进行统计汇总，比如求和、平均值等。比如下面的例子，我们可以计算飞机起飞延误不超过1小时的比例。

    .groups = "drop"

1.2.5 条件赋值 (Conditional transformations)

(1) if_else()

if_else()有4个参数，第1个是条件判断，第2个是条件判断为TRUE是返回的值，第3个是条件判断为FALSE时返回的值，第4个是条件判断为NA时返回的值。

x <- c(-3:3, NA)
if_else(x > 0, "+ve", "-ve", "???")
[1] "-ve" "-ve" "-ve" "-ve" "+ve" "+ve" "+ve" "???"
if_else(x == 0, "0", if_else(x > 0, "+ve", "-ve", "???"))
[1] "-ve" "-ve" "-ve" "0"   "+ve" "+ve" "+ve" NA

(1) case_when()

case_when()类似嵌套的if_else()语句，如果没有匹配到条件，会赋值NA。

x <- c(-3:3, NA)
case_when(
  x == 0 ~ "0",
  x < 0 ~ "-ve",
  x > 0 ~ "+ve",
  is.na(x) ~ "???"
)
[1] "-ve" "-ve" "-ve" "0"   "+ve" "+ve" "+ve" "???"
case_when(
  x < 0 ~ "-ve",
  x > 0 ~ "+ve"
)
[1] "-ve" "-ve" "-ve" NA    "+ve" "+ve" "+ve" NA

需要注意的是，case_when和if_else() 的输出结果需要时兼容的，例如输出的结果必需全是字符型或数值型，不能字符和数值两种形式掺杂。

2.3 算数运算和向量回收规则

算数运算符包括：+, -, *, /, ^，这些运算符可以用来对数值进行基本的算术运算，算数运算很简单，我们不会详细介绍。但需要注意的是，在进行算术运算时，如果两个向量的长度不相等，R会自动进行向量回收（vector recycling），R会重复较短的向量来匹配较长的向量。 通常情况下，我们希望回收单个数值（长度为1的向量），但R会对任何长度较短的向量执行回收操作。如果较长的向量的长度不是较短的向量长度的倍数，R会发出一个警告，提示回收可能导致不一致的结果。

x <- c(1, 2, 3, 4)
y <- c(10, 20)
x + y
[1] 11 22 13 24
x1 <- c(1, 2)
y1 <- c(10, 20, 3, 4)
x1 * y1
[1] 10 40  3  8
x2 <- c(1, 2)
y2 <- c(10, 20, 3)
x2 * y2
Warning in x2 * y2: longer object length is not a multiple of shorter object
length
[1] 10 40  3

回收规则同样应用于比较运算（>, <, ==等），会产生意外的结果如果使用==而不是%in%来检查一个值是否在一个向量中。

2.4 最小值和最大值（Min and max）

pmin()和pmax()函数可以用来计算两个或多个向量的元素级最小值和最大值。 需要注意的是min()和max()函数返回的是整个向量的最小值和最大值，会得到所有行的min和max值有一样的错误结果，而不是元素级的最小值和最大值。

df <- tibble(
  x = c(1, 2, 3),
  y = c(4, 1, NA)
)

df %>%
  mutate(
    min = pmin(x, y, na.rm = TRUE),
    max = pmax(x, y, na.rm = TRUE)
  )

2.6 对数运算（Logarithms）

常用的对数函数包括：log()（以e为底）, log10()（以10为底）, log2()（以2为底）。

2.7 四舍五入（Rounding）

round()是常用的四舍五入函数,round()函数会将数值四舍五入至整数。当然，可以用过digits来控制精度，round(x, digits)会将数值四舍五入至最接近的10^-n位数，所以digits=2表示四舍五入到最接近的0.01位。同理，若digits=-2示四舍五入到最接近的100位。

round(123.456)
[1] 123
round(123.456, digits = 2)  
[1] 123.46
round(123.456, digits = -2)
[1] 100

有个地方需要注意的是，round()函数在处理.5的情况时，采用的是”round to even”的规则，也就是当数值正好在两个整数的中间时，会将其四舍五入到最近的偶数。这是为了避免在大量数据中产生系统性的偏差，一半的.5会向上取整，一半会向下取整。

round(2.5)
[1] 2
round(4.5)
[1] 4

类似的函数还有ceiling()和floor()，ceiling()函数会将数值向上取整到最近的整数，而floor()函数会将数值向下取整到最近的整数。

ceiling(2.3)
[1] 3
floor(2.7)
[1] 2

同时，如果你想把一个整数四舍五入到某个数的倍数，可以先除以那个数，使用round()函数进行四舍五入，然后再乘以那个数。这在画图横坐标最大刻度计算时非常有用。

round(123.456 / 5) * 5
[1] 125
round(123.456 / 10) * 10
[1] 120

2.8 将数值分段（Cutting and binning）

cut()函数可以用来将数值分段，cut(x, breaks)函数会将数值x分成由breaks指定的区间，并返回一个因子变量，表示每个数值所属的区间。cut()函数还有一些其他参数，比如labels可以用来指定每个区间的标签。在范围外的数值会被标记为NA。同时可以通过include.lowest参数来控制是否将边界值包含在第一个区间中，默认情况下，cut()函数会将左边界包含在区间外，而将右边界排除在内。

x <- c(1, 2, 3, 4, 5, -1)
cut(x, breaks = c(0, 2, 4, 6))
[1] (0,2] (0,2] (2,4] (2,4] (4,6] <NA> 
Levels: (0,2] (2,4] (4,6]
z <- cut(x, breaks = c(0, 2, 4, 6), labels = c("Low", "Medium", "High"))
z
[1] Low    Low    Medium Medium High   <NA>  
Levels: Low Medium High
class(z)
[1] "factor"

2.9 累积与滚动汇总（Cumulative and rolling summaries）

cumsum()函数可以用来计算累积和，cumprod()函数可以用来计算累积积，cummin()函数可以用来计算累积最小值，cummax()函数可以用来计算累积最大值。这些函数在数据分析中非常有用，可以帮助我们理解数据的趋势和变化。

2.10 排序（Ranks）

min_rank()函数可以用来计算最小排名，row_number()函数可以用来计算行号，dense_rank()函数可以用来计算密集排名，percent_rank()函数可以用来计算百分位排名。这些函数在数据分析中非常有用，可以帮助我们理解数据的分布和相对位置。

# 当没有order_by参数时，first()、nth()和last()函数会按照数据的原始顺序来选择观测。也就是说，它们会选择每组中的第一个、第二个或最后一个观测，而不考虑任何特定的排序。
flights |>

3 字符串（Strings）

在程序中，你可以输入str_来查看字符串相关的函数，字符串相关的函数非常多，下面我们介绍一些常用的函数。 ## 3.1 创建字符串 ### 3.1.1 转义符 字符串中常包含单引号或双引号，可以使用转义字符\来表示这些特殊字符，或者使用不同类型的引号来包裹字符串。

backslash <- "\\"
single_quote <- "\'"
double_quote <- "\""
str_view(c(backslash,single_quote,double_quote))
[1] │ \
[2] │ '
[3] │ "

3.1.2 创建字符串函数

(1) str_c()

str_c()函数可以用来连接字符串，类似paste0()函数，但str_c()函数有一些额外的功能，比如可以指定分隔符、处理NA值等。

letters <- c("x", NA, "z")
str_c(letters, "-a")
[1] "x-a" NA    "z-a"
str_c(letters, letters)
[1] "xx" NA   "zz"
paste0(letters, "-a")
[1] "x-a"  "NA-a" "z-a" 
str_c(str_replace_na(letters, "y"), "-a")
[1] "x-a" "y-a" "z-a"

(2) str_glue()

str_glue()函数可以用来创建字符串，类似于Python中的f-string，可以在字符串中直接嵌入变量，非常方便。

df <- tibble(
  name = c("Alice", "Bob", "Charlie"),
  age = c(25, 30, NA)
)
df |> 
  mutate(
    message = str_glue("{name} is {age} years old.")
  )
# A tibble: 3 × 3
  name      age message                 
  <chr>   <dbl> <glue>                  
1 Alice      25 Alice is 25 years old.  
2 Bob        30 Bob is 30 years old.    
3 Charlie    NA Charlie is NA years old.

NA在str_c()和str_glue()中的处理方式不同，str_c()会将NA视为一个字符串，而str_glue()会将NA视为一个缺失值，并且在生成的字符串中会显示为NA。 如果你想显示文本{,}，可以使用双大括号来转义它们。

df <- tibble(
  name = c("Alice", "Bob", "Charlie"),
  age = c(25, 30, NA)
)
df |> 
  mutate(
    message = str_glue("{{{name} is {age} years old.}}")
  )
# A tibble: 3 × 3
  name      age message                   
  <chr>   <dbl> <glue>                    
1 Alice      25 {Alice is 25 years old.}  
2 Bob        30 {Bob is 30 years old.}    
3 Charlie    NA {Charlie is NA years old.}

(3) str_flatten()

str_flatten()函数可以用来将一个字符串向量连接成一个单一的字符串，可以指定分隔符。在summarize()中使用str_flatten()函数可以将一个分组内的字符串连接成一个单一的字符串，非常有用。

df <- tibble(
  group = c("A", "A", "B", "B"),
  value = c("x", "y", "z", "w")
)
df |> 
  group_by(group) |> 
  summarize(
    combined = str_flatten(value, collapse = ","),
    .groups = "drop"
  )
# A tibble: 2 × 2
  group combined
  <chr> <chr>   
1 A     x,y     
2 B     z,w
